# Milestone 7.5 — "Persona Wow" (proactive auto-pick across personas)

**Status:** Planning (plan-only 2026-05-11) — no code change yet.
**Position:** Between M7 (Lifecycle Wow, shipped) and M8 (Public Wow, planning).

---

## Context

MCP server `odoo-semantic` exposes 14 tools (`resolve_model`, `impact_analysis`, `find_override_point`, `api_version_diff`, ...) but AI clients (Claude Code, Claude.ai, Gemini, ChatGPT) **do not auto-pick** them in real sessions — users must explicitly name the MCP or paste the tool name. Three root causes were validated in a real session (2026-05-11):

1. **Docstrings describe WHAT, not WHEN.** No `TRIGGER`/`PREFER`/`SKIP` clauses → the routing model has nothing to match natural utterances against.
2. **No skill/agent layer.** When tools are not surfaced (untrusted directory, MCP still connecting, lazy-loaded), the client has no signal that the capability exists at all.
3. **Tech-language mismatch.** Non-technical personas (CEO, sales, marketing, BA) phrase questions in business language ("Can Odoo do X?", "How risky is the v18 upgrade?") which never reaches `resolve_model("sale.order")` because the language gap is too wide.

**Outcome wanted:** All five personas (CEO, dev, consultant, marketer, sales) can ask in natural language (EN + VN) and Claude / Gemini / ChatGPT automatically route through the right skill → the right MCP tool → the right output format for that persona.

### Research underpinning this plan

Two Sonnet subagents researched specs + persona design (2026-05-11). Key findings:

**Spec landscape (Anthropic, OpenAI, Google):**
- Strongest auto-pick signal across all three vendors is the **`description` field of a tool/skill** containing concrete WHEN clauses with natural utterances.
- Claude Code Skills + Sub-agents are top-tier (★★★★★) because their descriptions are surfaced in the system prompt every turn, even before MCP tools load.
- MCP routing is pure-description across vendors (Claude / OpenAI / Google ADK all bind via `description`).
- Claude Code Skill cap: 1,536 chars; budget 1% of context window.
- Trust dialog: project-scoped MCP servers require trust approval before tools surface; user-scoped servers bypass this — explains why `odoo-semantic` did not appear in untrusted directories in our session.

**Persona skill design:** 11 narrow skills + 2 router sub-agents (see [Skill × Persona table](#skill--persona-table) below). Each skill solves one concrete job-to-be-done. PREFER/SKIP clauses prevent overlap; Vietnamese `when_to_use` phrases are first-class (Viindoo team communicates in VN).

---

## Approach — Combo of two tracks

### Track 1 — Tool docstring TRIGGER blocks

Rewrite the 14 docstrings in `src/mcp/server.py` adding a `TRIGGER when: / PREFER over: / SKIP when:` block to each. Highest-leverage change: editing one file, **every client** (Claude, Gemini, ChatGPT) benefits because MCP spec routing depends solely on `description`.

### Track 2 — Claude Code plugin package

Build `dist/odoo-semantic-plugin/` bundling MCP config + 11 persona skills + 2 router sub-agents + an interactive setup command. Users install with one command; downstream we publish converted variants for Gemini Gems, OpenAI Custom GPTs, and Cursor.

The two tracks are **independent and parallelizable** — they can be released separately and a stop-point gate after Track 1 decides whether Track 2 is still warranted.

## Resolved decisions (user-confirmed 2026-05-11)

1. **Marketplace:** Viindoo self-host. Setup flow: `claude plugin marketplace add viindoo/claude-plugins` → `claude plugin install odoo-semantic`. Faster updates, no dependency on Anthropic review.
2. **Auth model:** `/odoo-semantic:connect` slash command prompts user for the API key (manual paste or Viindoo SSO link), then writes the entry into `~/.claude.json`. Plugin ships a template `.mcp.json` **without** a key — prevents leakage and rate-limit abuse.
3. **Persona docs locale:** EN canonical first, VN translation in a follow-up sprint via the existing `/translator` skill (T3.4b).

## Why M7.5 (not M8 sub-stream)?

M8 ("Public Wow") is about opening production to anonymous public traffic + landing site + admin from Internet — a deployment/marketing milestone. M7.5 is about **client-side adoption mechanics** (proactive auto-pick, persona skills, plugin distribution). The two are independent: M8 can ship without M7.5 (production opens regardless), and M7.5 can ship without M8 (persona skills work against the existing production endpoint). They are interleaved chronologically because both started planning the same day, but they should not be coupled.
