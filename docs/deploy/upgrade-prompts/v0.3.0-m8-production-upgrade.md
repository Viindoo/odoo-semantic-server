## Context — PR #86 đã làm gì (high-level)

v0.3.0 = M8 "Public Wow" — chuyển Web UI từ Jinja2 server-rendered HTML sang **Astro SSR (Node.js)** + FastAPI thành **pure JSON API**. Cộng thêm các thay đổi absorbed từ PR #87, #88 đã merge trước M8.

### Architecture delta

| Aspect | v0.2.0 (M7.5) | v0.3.0 (M8) |
|---|---|---|
| Process count | 2 Python services | 2 Python + 1 Node.js = **3 services** |
| MCP server | `127.0.0.1:8002` | `127.0.0.1:8002` (unchanged + P0 fix: PG pool init at startup) |
| Web UI backend | `127.0.0.1:8003` (FastAPI + Jinja2 HTML) | `127.0.0.1:8003` (FastAPI **JSON only**, no HTML) |
| Web UI frontend | served by FastAPI (Jinja2) | **NEW** `127.0.0.1:4321` (Astro Node.js SSR) |
| Nginx routing for `/admin/*`, `/` | → 8003 (HTML) | → **4321** (Astro). Nginx config phải swap atomically. |
| Nginx routing for `/api/*` | (internal only) | → **8003** (JSON, public via nginx) |
| Node.js | not required | **required ≥20** |
| pnpm | not required | **required ≥9** |
| Build step | none | `cd site && pnpm install --frozen-lockfile && pnpm build` (produces `site/dist/server/entry.mjs`) |
| PG migrations new | — | `0002_master_data_seed.sql` + `0003_profile_hierarchy.sql` + `9000_webui_users.sql` (if not yet applied) |
| Neo4j schema | nodes có 8 types | nodes có thêm `profile: list[str]` property (ADR-0016). Backfill optional — chỉ cần khi dùng `profile_name` filter. |
| Master data seed | manual | auto-runs on `python -m src.db.migrate` (idempotent `ON CONFLICT DO NOTHING`) |
| Web UI auth users | per-user via SSH-tunnel or hardcoded | bcrypt-hashed in `webui_users` table; **fresh M8 = empty table → admin login impossible until `create-webui-user admin` runs** |
| Critical env vars | `WEBUI_SESSION_SECRET`, `FERNET_KEY` | same + optionally `FASTAPI_BASE` cho split-tier |
| ADRs | up to 0013 | + **0014 Astro unified**, **0015 FastAPI pure JSON**, **0016 Profile hierarchy** |

### Bug fixes worth knowing

Trong quá trình develop M8, có 2 lớp bug được fix mà production cần aware:
- **datetime serialization 500**: `JSONResponse` với object chứa `datetime` raise TypeError → 500. Fix bằng `_json_safe()` helper ở boundary. Áp dụng vào dashboard stats, SSH keys list.
- **Astro 5.x `checkOrigin` security**: mutation fetch (POST/PUT/DELETE/PATCH) phải có `Content-Type: application/json` header, otherwise Astro reject với 403 "Cross-site form submission forbidden". Tất cả Astro pages trong M8 đã set header này. Production note: nếu có custom curl scripts hoặc external integration POST tới Astro endpoint, chúng phải set header.

## Rollback procedures

For each phase, document exact reverse:

- **Phase 8 (nginx)**: `sudo cp .../odoo-semantic-mcp.pre-v0.3.0.bak .../odoo-semantic-mcp && nginx -t && systemctl reload nginx`
- **Phase 7 (services)**: `systemctl stop odoo-semantic-astro && systemctl disable odoo-semantic-astro`
- **Phase 4 (migrations)**: NO yoyo down-migrations → restore PG dump: `dropdb + createdb + psql < backup.sql`
- **Phase 2 (code)**: `git checkout <previous tag>` + redeploy (nhưng schema đã migrate → cần rollback DB trước)

Note: schema rollback = PG dump restore. Không có incremental down path.

## Top 5 risks (Claude Code must call out explicitly)

1. **CRITICAL — Nginx + Astro non-atomic**: Nếu update nginx (route `/admin/*` → 4321) mà Astro chưa up → user thấy 502. Nếu Astro up mà nginx chưa update → user thấy JSON 401 thay vì login page. **Coordinate trong single maintenance window. Test 4321 trực tiếp trước khi reload nginx.**
2. **HIGH — `webui_users` empty sau migrate**: Admin UI permanently inaccessible until `create-webui-user admin` run. Block cutover until verified ≥1 row.
3. **HIGH — Migrate as root vs service user**: `seed_repos()` dùng `Path.home()` → root ghi `/root/git/...` paths invalid. Hard requirement: `sudo -u odoo-semantic -H`.
4. **MED — Sessions invalidate**: Nếu `WEBUI_SESSION_SECRET` chưa set hoặc đang dùng random per-process → service restart logout all admins. Set persistent secret trước upgrade.
5. **MED — Neo4j `profile_name` filter returns empty cho old data**: Backfill optional. Document gap cho admin nếu họ dự định dùng filter ngay sau cutover.

## Constraints + style guide

- **Đừng paste lại đoạn này** vào runbook. Runbook chỉ có commands + verify expected output cụ thể cho server này.
- **Substitute placeholders**: `<working dir>`, `<dsn>`, `<domain>` etc. phải resolve cụ thể từ Phase 1 investigation.
- **Mỗi step phải có verify expected output** — human biết step pass/fail liền, không cần guess.
- **Highlight HARD BLOCKERS** với 🚨 hoặc bold inline — không chôn trong giữa table.
- **No marketing copy** — chỉ ops content.
- **Tiếng Việt cho narrative**, English cho commands + technical terms — match style codebase này.
- **Nếu phát hiện anomaly** Phase 1 không cover (vd custom systemd unit names, monorepo layout khác, alternate DB host) → note clearly trong runbook + đề xuất manual verify thay vì assume.
