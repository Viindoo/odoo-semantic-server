## Failing queries (5 total)

| Query | Expected | Predicted | Why confused |
|-------|----------|-----------|--------------|
| "tính năng phê duyệt đơn hàng có trong Odoo CE không" | odoo-feature-check | none | Query lacks explicit TRIGGER phrase — no "does odoo have", "odoo có sẵn", or "check if" keyword; heuristic falls through. Fix: add "có trong odoo" as TRIGGER in skill description. |
| "scope of customization needed before we give the project estimate" | odoo-gap-analysis | odoo-customization-inventory | "scope of customization" matches both skills; `odoo-customization-inventory` pattern was checked first. Fix: add "estimate" as disambiguating TRIGGER for `odoo-gap-analysis`. |
| "tóm tắt tính năng mới Odoo 17 bằng ngôn ngữ kinh doanh cho khách hàng" | odoo-feature-highlights | odoo-version-diff | "tính năng mới Odoo 17" matches `odoo-version-diff` before `odoo-feature-highlights` in heuristic priority. Fix: add "ngôn ngữ kinh doanh" or "marketing" context to `odoo-feature-highlights` VI TRIGGERs. |
| "what Viindoo-specific modules are not available in standard Odoo CE or EE" | odoo-addon-diff | odoo-feature-check | "standard Odoo" and absence of explicit "CE vs EE" or "compare" phrase causes mis-route to `odoo-feature-check`. Fix: add "not available in standard" with CE/EE context to `odoo-addon-diff` TRIGGERs. |
| "does Odoo support barcode scanning in the warehouse — give me proof for the demo" | odoo-capability-proof | odoo-feature-check | Query starts with "does Odoo support" which is a strong `odoo-feature-check` TRIGGER; "give me proof" is not caught because "give me" precedes the TRIGGER phrase. Fix: add "give me proof" or "for the demo" as TRIGGER in `odoo-capability-proof`. |

**Root cause pattern:** All 5 misses are from overlapping TRIGGER phrases at skill boundaries — not from wrong skill design. Three confusions involve `odoo-feature-check` absorbing queries that contain "standard Odoo" or "does Odoo support" even when the intent is addon-diff or capability-proof. Recommended fixes are minor TRIGGER additions to the relevant `SKILL.md` description fields.

## Issues (P0/P1/P2)

No P0 or P1 issues. All personas pass ≥80% threshold.

### P2 — Minor TRIGGER phrase gaps (5 queries, non-blocking)

**P2-AR-001:** `odoo-feature-check` — Vietnamese query "có trong Odoo CE không" pattern not covered by TRIGGER list. Recommendation: add "có trong odoo" to description VI TRIGGER list.

**P2-AR-002:** `odoo-gap-analysis` vs `odoo-customization-inventory` — "scope of customization" phrase is ambiguous. Recommendation: add "project estimate" or "estimate customization effort" as a tie-breaking TRIGGER for `odoo-gap-analysis`.

**P2-AR-003:** `odoo-feature-highlights` VI — "tóm tắt tính năng mới" is shared with `odoo-version-diff`. Recommendation: add "ngôn ngữ kinh doanh" or "cho khách hàng" context to `odoo-feature-highlights` VI TRIGGERs.

**P2-AR-004:** `odoo-addon-diff` — Queries about Viindoo vs standard CE/EE without explicit "CE vs EE" phrasing fall through to `odoo-feature-check`. Recommendation: add "viindoo-specific modules" and "not available in standard" as TRIGGERs.

**P2-AR-005:** `odoo-capability-proof` — Queries starting with "does Odoo support X" + "give me proof" are captured by `odoo-feature-check` first. Recommendation: add "give me proof", "for the demo" as TRIGGERs in `odoo-capability-proof`.

These are heuristic-level gaps in the static proxy; actual Claude Code LLM routing should handle most of these correctly due to semantic understanding. Recommend re-testing these 5 queries in a live M8 session to confirm.

## Methodology notes

- **Heuristic priority order** (most to least specific): `odoo-objection-handler` → `odoo-risk-overview` → `odoo-customization-inventory` → `odoo-override-finder` → `odoo-deprecation-audit` → `odoo-version-diff` → `odoo-gap-analysis` → `odoo-feature-check` → `odoo-feature-highlights` → `odoo-addon-diff` → `odoo-capability-proof`
- `odoo-objection-handler` must be checked first because it shares "doesn't support" / "cannot" phrases with several other skills.
- `odoo-gap-analysis` must be checked before `odoo-feature-check` because gap analysis queries often contain "standard Odoo" phrasing.
- Query source: 30 queries reused/adapted from `tests/test_skill_disambiguation.py`; 95 new queries generated from TRIGGER phrases in each `SKILL.md` description and persona doc sample questions (`docs/personas/`). EN/VI balance: 63 EN / 62 VI.
