### [P2] `suggest_pattern` returns "no patterns indexed" on production — 2026-05-14

- **Severity:** P2 (operational gap, not a code regression).
- **Symptom:** `suggest_pattern(intent="...", odoo_version="17.0")` → `no patterns indexed. Run: python -m src.indexer.seed_patterns`. Reproduces for any intent/version.
- **Root cause:** `seed_patterns` step not run on production server post-deploy; `_SeedMeta` sentinel absent.
- **Impact:** 1/14 MCP tool non-functional in production. `find_examples` (vector search) unaffected.
- **Suggested fix:** Admin action: `python -m src.indexer.seed_patterns` on production. After that, `_SeedMeta` sentinel auto-skips re-embed on future runs (per ADR-0007 auto-reseed).
- **Owner:** Production admin.
- **Tracked in:** TASKS.md M8 backlog as M7.5-P2-SEED.

### [P1] CoreSymbol index missing `name_get` for 16.0 and 17.0 — Batch 1 — 2026-05-14

- **Affected:** `mcp__odoo-semantic__lookup_core_api`, `mcp__odoo-semantic__api_version_diff`
- **Symptom:** Both tools return "not found" for `name_get` symbol on Odoo 16.0 and 17.0. This is a well-known ORM method deprecated in 17.0 (replaced by `display_name`/`_compute_display_name`).

- **Severity reasoning:** P1 — index gap for canonical deprecated ORM method makes deprecation-audit personas blind to a major upgrade gotcha. Misleads developers about whether the API ever existed.
- **Suggested fix:** Re-run `python -m src.indexer index-core` for 16.0 and 17.0. Verify `name_get` is captured from `odoo/models.py` parser output. Check `_resolve_core_paths()` for 17.0 against ADR-0005.
- **Owner:** Indexer / Core parser
- **Blocks:** §6 tool 7, §6 tool 8
