## New issues / anomalies

### P2 (confirmed across all 3 runs) — `suggest_pattern`: PatternExample nodes not seeded on production

- **Symptom:** `suggest_pattern(...)` returns `no patterns indexed. Run: python -m src.indexer.seed_patterns` regardless of intent or version.
- **Root cause:** `seed_patterns` step not executed after server deployment; `_SeedMeta` sentinel absent.
- **Impact:** `suggest_pattern` non-functional in production. Other 13 MCP tools unaffected. `find_examples` (vector search) works normally.
- **Severity:** P2 — operational gap, not a code regression.
- **Action:** Admin task on production server: `python -m src.indexer.seed_patterns`. Tracked in TASKS.md M8 backlog.

### Informational (non-blocking)

1. **`cli_help` curation banner**: Returns `ℹ Spec data v17.0 pending curation — limited results.` prefix. Flag payload itself is correct (default=8072, type=int). Banner is a heuristic about overall v17 CLI spec coverage, not a per-flag warning.
2. **`api_version_diff` 16.0 gap**: Shows `name_get` as "added in 17.0" because v16 core symbols haven't been re-indexed yet. Per PR #84 backlog (Tier 2 v9–v16/v18/v19 coverage). Tool returns cleanly without error.
3. **Cross-repo indexing confirmed**: `resolve_model(account.move)` Extended-by list includes `l10n_vn_edi_viettel` (Viindoo localisation), proving multi-repo merge is active alongside upstream Odoo CE.
