## Issues found

### P2 ÔÇö suggest_pattern: PatternExample nodes not seeded on production server

- **Severity:** P2 (degraded feature, not broken infra)
- **Symptom:** `suggest_pattern` returns "no patterns indexed. Run: python -m src.indexer.seed_patterns" for all queries, regardless of version.
- **Root cause:** `seed_patterns` was apparently not run after server deployment. The `_SeedMeta` sentinel node in Neo4j likely absent.
- **Impact:** `suggest_pattern` tool is non-functional on production. `find_examples` (vector search) works normally.
- **Recommendation:** Admin action: `python -m src.indexer.seed_patterns` on the production server. This is an operational gap, not a code regression.

### Note ÔÇö Documentation correction needed (argument name drift)

The task brief used stale argument names (`symbol_name`, `module_name`, `query` for suggest_pattern). Live server schema uses `name`, `name`, `intent` respectively. Verification scripts or runbooks that hardcode old names will fail. Consider adding a `tools/list` discovery step to future verification runbooks.
