## Issues (P0/P1/P2)

### P1 — Ollama SSL / embedding service down (tools 5, 12)

**Affects:** find_examples, suggest_pattern (both embedding-dependent tools)
**Error:** `RuntimeError: Qwen3Embedder failed after 3 attempts: <urlopen error [SSL: CERTIFICATE_VERIFY_FAILED] certificate verify failed: self-signed certificate (_ssl.c:1010)>`
**Root cause:** The MCP server cannot reach Ollama over HTTPS due to SSL certificate verification failure against a self-signed cert. Either Ollama is running behind a reverse proxy with a self-signed cert that the server doesn't trust, or the Ollama URL scheme is misconfigured (should be `http://` not `https://`).
**Impact:** 2 tools fully non-functional. Vector-search features unavailable to all clients.
**Fix:** Check `OLLAMA_BASE_URL` in server config — if pointing to `https://localhost:11434`, change to `http://localhost:11434`. Or add self-signed CA to trusted store.

### P1 — CoreSymbol catalogue missing name_get (tools 7, 8)

**Affects:** lookup_core_api, api_version_diff
**Symptom:** `name_get` returns "not found" for both 16.0 and 17.0. This symbol is a well-known Odoo ORM method that was deprecated in 17.0 (replaced by `display_name`/`_compute_display_name`).
**Root cause:** CoreSymbol index for this method was not ingested — likely a path or parser coverage gap in `parser_odoo_core.py`.
**Impact:** Tools return "not found" for known deprecated symbols, which is misleading (clients may think the API never existed).
**Fix:** Re-run `index-core` for 16.0 and 17.0; verify `name_get` appears in `odoo/models.py` parse output. Check `_resolve_core_paths()` for 17.0.

### P1 — CLICommand catalogue missing --gevent-port (tool 11)

**Affects:** cli_help
**Symptom:** `cli_help('server', '--gevent-port', '17.0')` returns flag not found. The flag is documented in Odoo 17 and appears in configs (replaces `--longpolling-port`).
**Root cause:** CLICommand index gap — `--gevent-port` was either not parsed from `odoo/cli/server.py` or not stored for version 17.0.
**Fix:** Re-run `index-core` CLI parsing for 17.0; check that `--gevent-port` argparse definition is captured.

### P2 — lint_check fuzzy matcher misses canonical UserError formatting rule (tool 10)

**Affects:** lint_check
**Symptom:** `raise UserError('Hello %s' % name)` returns 0 violations. This is a canonical Odoo lint violation (string formatting in UserError/ValidationError — should use lazy `_()` or pass args separately).
**Root cause:** V0 fuzzy token-overlap matcher requires ≥2 significant token overlap between rule message and submitted code. The LintRule catalogue may lack the specific rule, or the token overlap threshold is not met.
**Impact:** First-pass lint screen misses a common code quality issue. Tool is labeled "V0 fuzzy — verify manually" so this is expected behavior per spec, but the rule should be in the catalogue.
**Fix:** Audit LintRule catalogue for UserError/ValidationError string formatting rule; add if missing. Track as quality debt for V1 lint matcher.
