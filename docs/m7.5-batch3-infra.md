## Issues

### P1.1 — HSTS Header Missing

**Endpoint:** `/health` (and likely all endpoints)

**Observed:** No `Strict-Transport-Security` header in response.

**Expected per §1.1:** `Strict-Transport-Security: max-age=31536000; includeSubDomains`

**Impact:** Browsers do not enforce HSTS; clients can fall back to HTTP on next visit. TLS security posture weakened.

**Remediation:**
1. Verify Nginx config contains:
   ```nginx
   add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
   ```
2. Reload: `systemctl reload nginx`
3. Retest: `curl -sI https://odoo-semantic.viindoo.com/health | grep Strict`

**Owner:** Infra / Nginx

---

### P1.2 — Web UI Not Accessible (/admin returns 404)

**Endpoint:** `/admin/repos`, `/admin/login`

**Observed:** HTTP/2 404 (not 302 redirect to login)

**Expected per §10.2:** HTTP/2 302 Location: `/admin/login`

**Impact:** Web UI inaccessible for admin tasks; pre-launch checklist §10 cannot be verified.

**Remediation:**
1. Check Web UI process: `systemctl status odoo-semantic-webui` or `docker ps | grep webui`
2. If not running: start service
3. Verify Nginx upstream proxy configuration:
   ```nginx
   upstream webui_backend {
       server 127.0.0.1:8003;
   }
   location /admin/ {
       proxy_pass http://webui_backend;
   }
   ```
4. Check Web UI logs for startup errors
5. Retest: `curl -sI https://odoo-semantic.viindoo.com/admin/repos`

**Owner:** Infra / Web UI deployment
